這些程式碼應該放在哪裡？

如果你來自於純 PHP 的背景（沒有使用現代框架），你可能習慣於將程式碼放在網頁伺服器的文檔根目錄下（如 /var/www）。使用 Django 時，你不需要這麼做。將任何 Python 程式碼放在網頁伺服器的文檔根目錄下是不安全的，因為這可能會讓人通過網絡查看你的程式碼。

將你的程式碼放在文檔根目錄外部的某個目錄，如 /home/mycode。

創建投票應用

現在你的環境 —— 一個 “項目” —— 已經設定好了，你就可以開始工作了。

每個你在 Django 中寫的應用都是遵循一定慣例的 Python 包。Django 附帶了一個工具，可以自動生成應用的基本目錄結構，讓你可以專注於寫程式碼而不是創建目錄。

項目與應用有什麼區別？

應用是一個做某事的網路應用程式 —— 例如，一個 Blog 系統、一個公共記錄的資料庫或一個小型的投票應用。一個項目是一個特定網站的配置和應用的集合。一個項目可以包含多個應用。一個應用可以在多個項目中使用。

你的應用可以存放在你的 Python 路徑 <tut-searchpath> 的任何地方。在這篇教學中，我們將在與 manage.py 文件相同的目錄中創建我們的投票應用，這樣它就可以作為自己的頂級模組被導入，而不是作為 mysite 的一個子模組。
